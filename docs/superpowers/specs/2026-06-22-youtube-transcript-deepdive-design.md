# CC Daily — 動画コーナー「🎬 今日の1本」設計書

> 対象リポ: `cc-daily`　／　作成: 2026-06-22　／　承認: 桜口さん（頻度=良い日だけ・対象=海外中心＋日本も）
>
> **この文書は方針転換を含む。** 当初は「動画の文字起こしを自動取得して深掘り（方式B）」を設計したが、実証で全滅（後述）。桜口さんの判断で **「文字起こしはしない。AIは目利きに徹し、見るべき動画のリンク＋見どころを毎朝1本渡す」キュレーション方式**に転換した。

## 目的（一言）

CC Daily に **「🎬 今日の1本」** コーナーを足す。AIが直近7日の話題動画から「今日見るならこれ」を1本選び、**誰が・何を・見どころ・所要時間・リンク**だけ渡す。文字起こしして勉強するかは桜口さんが手元で判断する（AIは文字起こししない）。

## なぜ文字起こし（方式B）をやめたか — 2026-06-22 実証

号生成AI（クラウドRoutine・Sonnet・**WebFetchのみ**）で第三者文字起こしサービスを試験：

| サービス | 結果 |
|---|---|
| youtubetotranscript.com | ❌ 403 Forbidden（botブロック） |
| kome.ai / notegpt.io / tactiq.io | ❌ フォームUI・JS必須でWebFetch不可 |
| YouTube内部API（timedtext） | ❌ 空白（セッション必要） |

→ **現状の実行環境では文字起こしの自動取得は不可能**。無理に追うより、リンクと見どころのキュレーションに振った方が確実で、桜口さんの「リンクをくれれば自分で見る」という使い方にも合う。（将来 Playwright MCP が使える環境が来たら方式Bを復活できるよう、候補リストは `sources.json` に残す。）

## 確定した要件（2026-06-22 桜口さん回答）

| 項目 | 決定 |
|---|---|
| 頻度 | **良いネタがある日だけ**（0本の日があってよい・毎日固定にはしない） |
| 対象 | **海外中心＋日本も**。海外の定番（IndyDevDan / Cole Medin / Fireship / Matt Berman / AI Engineer）＋日本のAI/開発系（Claude Code・AI活用を分かりやすく実演する人）。経営者・非エンジニア向けを優先 |
| 内容 | リンク＋「誰が・何を・見どころ・所要時間」。**文字起こしはしない**（リンクと概要のみ＝WebSearchで確実に取れる） |
| 位置 | 号の**最下段に固定**（毎日同じ場所＝習慣化）。`categories` 配列の最後に置く |

## 設計：既存の仕組みにそのまま乗る（フロント変更ゼロ）

`app.js` の `renderIssue()` は **`categories` 配列を順番通りに描画する汎用ロジック**。各 item は `itemCard()` で描画される（title・trust・summary・ideas・url）。

→ **新コーナーは「号JSONの `categories` の最後に1ジャンル足すだけ」で実現する。`app.js` / `styles.css` / `index.html` は一切変更しない。** `validateIssue.js` も既存スキーマで通る（key を見ない汎用検証）。

```
categories: [ newword, howto, insight, insider, official, 👉 video（最後＝最下段） ]
```

### 号スキーマ（既存 item を流用・新フィールド不要）

`categories` の最後に追加するジャンル：

```jsonc
{
  "key": "video",
  "label": "🎬 今日の1本",
  "items": [
    {
      "title_ja": "発信者名｜動画のテーマ（例: Cole Medin｜上級者のClaude Code術）",
      "summary_ja": ["見どころ1（何を実演/解説しているか）", "見どころ2（塾運営・AI活用にどう効くか）", "見どころ3（誰向け・前提）"],
      "source_date": "動画の公開日（号の日付から7日以内）",
      "url": "https://www.youtube.com/watch?v=...",
      "trust": "🟨",
      "trust_reason": "本人チャンネルの実在動画をWebSearchで確認。本編内容は未視聴のため🟨（リンクと概要のみ）",
      "tags": ["アプリ開発", "自動化"],
      "ideas": ["関連してこの発信者の○○も見る価値あり 等（1つ以上）"],
      "try_hint": "約○分。まず最初の5分だけ見て、続きを見るか判断する 等"
    }
  ]
}
```

`itemCard()` の固定ラベル（「こんなことができます」「💡 ほかにもこんな使い方」「原文を開く↗」）は動画でも意味が通る範囲なので、今回はフロントを変えない。動画専用ラベルが欲しくなったら別案件。

## ガードレール（落とし穴対策）

1. **本人チャンネルの実在動画を確認** → WebSearchで発信者の本人チャンネル/動画URLを特定してから載せる。二次まとめ・切り抜き経由の不確かなものは🟥にするか載せない。
2. **trust は🟨止まり** → 本編は未視聴（リンクと概要のみ）なので緑にしない。`trust_reason` 必須。
3. **7日ルール厳守** → 動画公開日が号の日付から7日以内のものだけ。
4. **重複回避** → 過去号で出した動画URL・同じ話題は出さない。
5. **良い日だけ・0本OK** → これはという動画が無い日は video ジャンルを入れない（無理に作らない）。
6. **固有名詞** → タイトル/概要の Claude・Codex・Anthropic・MCP 等は公式名で表記。

## 非対象（YAGNI）

- 動画の文字起こし自動取得（方式B・断念。将来 Playwright MCP 環境で復活余地）。

- 動画専用カードUI（`itemCard` のラベル分岐。既存流用で足りる）。
- ポッドキャスト・カンファレンス長編の扱い（別案件）。
