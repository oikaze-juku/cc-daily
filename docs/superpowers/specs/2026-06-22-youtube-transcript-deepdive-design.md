# CC Daily — YouTube 文字起こし深掘り 設計書

> 対象リポ: `cc-daily`　／　作成: 2026-06-22　／　承認方式: 桜口さん①（spec化→ROUTINE/sources改修まで）

## 目的（一言）

CC Daily の動画ソースを「予告編しか見ていない状態（`type: search`）」から「**本編の文字起こしを読んで、一番おいしい実例を1つ切り出す**」状態に引き上げる。塾長が好む「実演系の具体ノウハウ」を毎朝の号で拾えるようにする。

## 背景・課題

現状の `video` セクション（IndyDevDan / Cole Medin / Fireship / Matt Berman / AI Engineer）は `type: search`。検索スニペットと二次まとめしか拾えず、動画の中で実演している**具体的な手順・実例**まで届かない。塾長の質問「有益な動画の文字起こしからネタを拾えないか」への回答がこの設計。

## 確定した要件（2026-06-22 桜口さん回答）

| 項目 | 決定 |
|---|---|
| 対象 | **海外の現状5人を軸**（IndyDevDan / Cole Medin / Fireship / Matt Berman / AI Engineer）。日本勢・教育系チャンネルは今回は見送り（将来拡張） |
| 頻度 | **良いネタがある日だけ**（件数固定なし・0本の日があってよい・任意） |
| 取得方式 | **方式B（第三者文字起こしサービス経由 WebFetch）を主軸**。全滅したら方式C（検索）にフォールバック |

## 設計概要：3段ハイブリッド

毎朝「全動画の字幕を取る」のは無人実行だと破綻する。よって「当たりをつけて1本だけ本編を読む」方式にする。

```
C段（当たりをつける）   WebSearch で海外5人の直近7日の動画から「これは」という1本を特定
        ↓
B段（本編を読む）        その動画URL→文字起こしサービスへ WebFetch して本編テキストを取得
        ↓
抽出（名場面を切る）      文字起こしから「一番おいしい実例・手順」を1つ抽出して記事化（🟨・動画URL併記）
        ↓
失敗時フォールバック      B段が全滅したら方式C（検索スニペット要約＝現状動作）に落として号は止めない
```

## 方式B：文字起こしサービスの候補とフォールバック順

号生成AI（クラウドRoutine・Sonnet）が **WebFetch だけ**で完結させる（新スクリプト・npm 導入は不要）。動画ID（`youtube.com/watch?v=◯◯` の `◯◯`）を渡してテキスト化する。下記を上から順に試し、**読めるテキストが返った最初のもの**を使う。

1. `https://youtubetotranscript.com/transcript?v=<VIDEO_ID>` （URLパラメータ式で WebFetch と相性が良い・第一候補）
2. `https://kome.ai/tools/youtube-transcript-generator` （動画URLを渡す）
3. `https://notegpt.io/youtube-transcript-generator`
4. `https://tactiq.io/tools/youtube-transcript`

**全滅したら**：その日は方式C（検索スニペット要約）にフォールバック。号生成は絶対に止めない。

> ⚠️ どのサービスが実際に WebFetch で取れるかは環境依存。**実証（次ステップ②）で第一候補を確定**する。spec 段階では「候補を順に試し、ダメなら C」を契約とする。

## データ（号スキーマは既存流用・変更なし）

`scripts/validateIssue.js` の現行スキーマで文字起こしitemはそのまま通る。新フィールド不要。

| フィールド | 文字起こしitemでの値 |
|---|---|
| `url` | **動画URL**（`https://www.youtube.com/watch?v=...`） |
| `source_date` | **動画の公開日**（号の日付から7日以内） |
| `trust` | **🟨**（自動字幕ベースのため緑にはしない） |
| `trust_reason` | 「動画本編の文字起こしから要約。自動字幕のため固有名詞は公式名に補正済み。出典は動画URL」 |
| `summary_ja` | 3行（技術→手間→効果） |
| `ideas` | 2つ（別の使い方） |
| `title_ja` | 発信者名を込める（例「IndyDevDan 実演：Claude Code で○○を自動化」） |

## ガードレール（落とし穴対策・盛らない）

1. **自動字幕の誤字** → 固有名詞は公式名に必ず補正（Claude / Codex / Anthropic / MCP 等）。怪しい数値・主張はそのまま載せず🟨止まり。
2. **全文転載禁止** → 要点＋出典URLの「引用の範囲」に留める。文字起こし全文は号に載せない。
3. **7日ルール厳守** → 動画公開日が号の日付から7日以内のものだけ。
4. **重複回避** → 過去号で扱った動画URL・同じ話題は出さない。
5. **ブロック時** → サービスが取れない日は静かに方式Cへ。エラーで号を落とさない。

- 手順3の `video` の扱いに「`type: transcript`：C段で1本特定→B段で文字起こし→名場面抽出」を追記。
- 手順4の深掘りに「動画deep-dive（任意・良いネタがある日だけ）」のフローを追記。

## 受け入れ基準

- [ ] ROUTINE/sources に transcript フローが明記され、号生成AIが「良い動画があれば1本深掘り／無理ならCへ」を判断できる。
- [ ] 文字起こしitemが既存スキーマで `validateIssue` を通る（trust=🟨・source_date=公開日・ideas≥1・summary 3行・url=動画）。
- [ ] 方式B全滅でも号生成が止まらない（Cフォールバックが明記）。
- [ ] （別ステップ②）実証で1本、実際に WebFetch で文字起こしが取れることを確認し第一候補を確定。

## 非対象（YAGNI・今回やらない）
